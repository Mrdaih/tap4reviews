# Tap4Reviews

NFC-powered Google review cards for UAE restaurants. Built with Next.js 14 (App Router), Tailwind, Supabase, and Stripe.

## Stack

- Next.js 14 + TypeScript (App Router)
- Tailwind CSS 3
- Supabase (Postgres + Auth)
- Stripe Checkout
- Resend for transactional email
- Framer Motion + Recharts for animations and charts
- MDX-backed blog

## Local development

```bash
npm install
cp .env.example .env.local   # fill in Supabase + Stripe + Resend keys
npm run dev
```

The site runs on http://localhost:3000.

## Supabase setup

1. Create a Supabase project.
2. Open the SQL editor and run `supabase/schema.sql` — it creates `profiles`, `cards`, `taps`, `orders`, and `posts` with row-level security.
3. In Authentication → Providers, enable Email + Google OAuth. For Google, whitelist `https://tap4reviews.com/auth/callback` and `http://localhost:3000/auth/callback`.
4. Copy the project URL, anon key, and service role key into your env file.

## Stripe setup

1. Create a Stripe account (AED currency).
2. Copy the live/test secret and publishable keys into `.env.local`.
3. Deploy the site, then create a webhook endpoint pointing at `/api/webhook/stripe` and put the signing secret into `STRIPE_WEBHOOK_SECRET`.
4. Products are defined in `lib/stripe.ts` — prices can be edited there without touching the DB.

## Deployment

### Option A: Vercel (recommended)

1. Push to GitHub.
2. Import the repo in Vercel.
3. Set environment variables (see above).
4. Add the domain `tap4reviews.com` in Vercel Project → Domains.
5. Point your DNS:
   - `A @` → `76.76.21.21` (Vercel) or the IP Vercel provides.
   - `CNAME www` → `cname.vercel-dns.com` (Vercel handles the redirect to apex).
6. Vercel handles SSL, CDN, and edge functions automatically.

Add a Google Search Console TXT record in DNS: `TXT @ "google-site-verification=..."`.

### Option B: cPanel (static export fallback)

> ⚠️ Static export disables API routes, the dashboard, and anything that requires a server. Use this only if you cannot use Vercel.

1. Add `output: 'export'` to `next.config.js`.
2. Run `npm run build`. Next writes the static site to `/out`.
3. Upload the contents of `/out` to `public_html` on your cPanel host.
4. The included `public/.htaccess` enables HTTPS redirect, www → apex, clean URLs, Gzip, and cache headers. Copy it to `public_html/.htaccess`.
5. Remove or replace `/api`, `/dashboard`, and `/login` routes since they require a Node server.

### DNS quick reference

| Record | Host  | Value                        | Purpose                 |
| ------ | ----- | ---------------------------- | ----------------------- |
| A      | @     | (Vercel / host IP)           | Apex domain             |
| CNAME  | www   | cname.vercel-dns.com         | www redirect            |
| TXT    | @     | google-site-verification=... | Search Console          |
| MX     | @     | (your email provider)        | Custom-domain email     |

## Project structure

```
app/
├── (marketing)/          # Public pages
├── (dashboard)/          # Authenticated dashboard
├── api/                  # Route handlers (tap, checkout, webhooks, etc.)
├── auth/callback         # Supabase OAuth callback
├── login                 # Auth entry
├── robots.ts             # /robots.txt generated by Next
├── sitemap.ts            # /sitemap.xml generated by Next
└── layout.tsx
components/
├── shared/               # Navbar, Footer, WhatsApp button, structured data
├── ui/                   # Section, Accordion, TapDemo
├── marketing/            # Contact form, legal shell
└── dashboard/            # Cards list, chart, QR, settings
content/blog/             # MDX blog posts
lib/
├── supabase/             # Browser, server, middleware Supabase clients
├── site.ts               # Site-wide constants
├── stripe.ts             # Stripe client + product catalog
├── posts.ts              # MDX blog loader
└── utils.ts              # cn(), formatAED(), whatsappLink()
public/
├── .htaccess             # cPanel config
└── llms.txt              # AI crawler summary
supabase/schema.sql       # Database schema
```

## Post-deployment checklist

- [ ] Supabase schema deployed and RLS verified.
- [ ] Stripe test purchase completes and appears in `orders`.
- [ ] `/api/tap/<serial>` redirects to Google review URL and logs a row in `taps`.
- [ ] Google Search Console DNS verification passed.
- [ ] Google Analytics 4 installed (set `NEXT_PUBLIC_GA_ID`).
- [ ] WhatsApp number in `.env` points to a live support line.
- [ ] Favicon and OG images uploaded.
- [ ] At least one card seeded per demo account.
